# عون — تقرير التدقيق الشامل وخطة التحسينات

> مراجعة كاملة للمشروع من ناحيتي **التصميم/تجربة الاستخدام** و**البرمجة/الأمان/الأداء**.
> كل ملاحظة موثّقة بمسار الملف ورقم السطر. التاريخ: 2026-07-13.

**الحكم العام:** الفكرة والهوية البصرية («الضوء الذي ينمو»، الوردة السباعية، الإزهار اليومي) مميزة فعلاً وفوق مستوى أغلب تطبيقات العادات. نظام الحركة (Motion) ممتاز. لكن تحت هذا الجمال تنفيذ غير منضبط: لا يوجد سلّم خطوط حقيقي، تدرّج العلامة مكرر بنسختين مختلفتين، نظامان للزوايا، ألوان hex مكتوبة يدوياً تتجاوز الـtokens، أهداف لمس أصغر من المعيار، لا حالات تركيز للوحة المفاتيح، وحذف بلا تأكيد. ومن جهة الكود: ثغرتان حرجتان في الأمان، وخطأ جوهري في المناطق الزمنية، وغياب كامل للتحقق من المدخلات.

## الجزء الأول: أخطاء حرجة يجب إصلاحها فوراً 🔴

### 1.1 سرّ الجلسات له قيمة افتراضية علنية — حرج
`src/lib/auth.ts:10`

```ts
process.env.AUTH_SECRET || "dev-secret-change-me"
```

لو نُشر التطبيق بدون ضبط `AUTH_SECRET` فكل الجلسات موقّعة بنص معروف للجميع، وأي شخص يستطيع تزوير كوكي `aoun_session` لأي `userId` والاستيلاء على أي حساب.

**الإصلاح:** احذف القيمة الافتراضية وارمِ خطأ عند الإقلاع في الإنتاج:

```ts
const secret = process.env.AUTH_SECRET;
if (!secret || secret === "dev-secret-change-me") {
  if (process.env.NODE_ENV === "production")
    throw new Error("AUTH_SECRET مطلوب في الإنتاج");
}
```

### 1.2 حماية مسار الـcron تعطّل نفسها — حرج
`src/app/api/cron/reminders/route.ts:11`

```ts
if (secret && secret !== "change-me-please" && auth !== `Bearer ${secret}`) { ... 401 }
```

إذا كان `CRON_SECRET` غير مضبوط أو ما زال بالقيمة الافتراضية، يُتخطى الفحص كلياً ويصبح المسار **عاماً**: أي شخص يستطيع إطلاق إشعارات Push لجميع المستخدمين ومسح الجدول كاملاً متى شاء.

**الإصلاح:** اعكس المنطق ليكون fail-closed — إن لم يوجد سرّ صالح، أعد 401 دائماً.

### 1.3 المنطقة الزمنية مخزّنة لكنها مُهملة — عالٍ (أهم خطأ منطقي)
- `src/lib/habits.ts:116` → `const tz = "Asia/Riyadh";`
- `src/lib/analytics.ts:49` → `const tz = "Asia/Riyadh";`

المخطط يخزّن `User.timezone`، والإعدادات تسمح بتغييرها، و`reminders.ts` يستخدمها بشكل صحيح — لكن اللوحة والتحليلات تحسبان «اليوم» بتوقيت الرياض دائماً. مستخدم في نيويورك يرى تاريخ الترويسة بيومه المحلي (لأن `dashboard/page.tsx` يستخدم توقيته) بينما تُسجَّل إنجازاته على يوم الرياض — انحراف يصل 8 ساعات، وتتعارض التذكيرات مع اللوحة حول «أي يوم نحن فيه».

**الإصلاح:** حمّل `user.timezone` في `getDashboard` و`getAnalytics` ومرّره لكل حسابات `dateKey`.

### 1.4 لا يوجد أي تحقق من المدخلات (Validation) — عالٍ
لا توجد مكتبة تحقق (zod أو غيرها) في المشروع إطلاقاً. الأجسام تُحوَّل بـ`as` فقط:

- `/api/habits` POST: يفحص `title` فقط؛ `frequency` و`weekdays` و`colorKey` و`scheduledAt` و`microSteps` و`reminderOffsetMin` تُكتب للقاعدة بلا فحص. يمكن إرسال `frequency: "hourly"` أو `weekdays` تالفة فيختل `isDueOn`.
- `/api/habits/[id]` PATCH: يطبّق الحقول حرفياً (`habits.ts:307-320`).
- `/api/user` PATCH: يقبل أي نص كـ`timezone` — قيمة IANA غير صالحة سترمي استثناء لاحقاً داخل `Intl.DateTimeFormat`.
- `/api/onboarding`: يفحص أن المصفوفة غير فارغة فقط.

**الإصلاح:** أضف طبقة zod مشتركة (أو حراس صريحين: enum للتكرار، `COLOR_KEYS` للون، regex `HH:mm` للوقت، فحص IANA للمنطقة).

### 1.5 مسارات API مكشوفة — عالٍ
- `/api/refine` **بلا مصادقة وبلا rate limiting** (`src/app/api/refine/route.ts`). عندما يكون المزوّد `http` فكل نداء يستهلك مفتاح LLM المدفوع — يمكن لمجهول حرق ميزانيتك.
- `/api/push/subscribe` **DELETE بلا مصادقة** (السطور 37-47): يحذف بأي `endpoint` يُرسل — أي شخص يعرف endpoint يستطيع إلغاء اشتراك جهاز مستخدم آخر.
- **POST upsert** (السطور 19-28): يعيد تعيين `userId` عند التعارض — مستخدم خبيث يستطيع «خطف» اشتراك مستخدم آخر.
- **لا يوجد rate limiting في المشروع كله**: تسجيل الدخول قابل للتخمين بالقوة الغاشمة.

### 1.6 حذف العادة بلا تأكيد وبلا تراجع — عالٍ (UX)
`HabitCard.tsx:230-237` و`ReviewStep.tsx:94-101`: «حذف العادة» ينفَّذ فوراً ويمحو تاريخ الـstreak كاملاً بلمسة خاطئة واحدة.

**الإصلاح:** حوار تأكيد + toast «تم الحذف — تراجع» بمهلة 5 ثوانٍ.

### 2.4 نظام تجربة الاستخدام

1. **أهداف اللمس أصغر من المعيار.** زر الإكمال — الزر الأكثر استخداماً في التطبيق — حجمه **26×26px** (`HabitCard.tsx:97`) والمعيار 44px (Apple) / 48px (Material). حوامل الألوان 32px، الإيموجي 36px. → كبّر منطقة اللمس لـ≥44px (يمكن إبقاء الشكل المرئي أصغر مع padding شفاف).
2. **نظام Toast موحّد.** حالياً التغذية الراجعة مبعثرة: `RewardPop` في اللوحة، «تم الحفظ ✓» inline في الإعدادات، «أرسلنا إشعاراً ✓» في التنبيهات. → مكوّن `<Toast>` واحد يخدم الحفظ والحذف-مع-تراجع والأخطاء.
3. **حالات التحميل كلها `"…"`** (AuthForm، SettingsClient، NotificationsToggle، «…يُضاف»، «…نُجهّز الصورة») → spinner موحّد + skeletons للوحة.
4. **الشريط السفلي (`BottomNav.tsx`):** جميل لكن (أ) لا يوجد زر «إضافة» فيه — الإضافة مدفونة كبطاقة متقطعة آخر القائمة؛ النمط الصحيح لتطبيق عادات هو **FAB مركزي** في الشريط. (ب) الإعدادات تصلها من الشريط **ومن** ترس اللوحة — احذف الترس. (ج) الحالة النشطة بلا مؤشر متحرك.
5. **ترويض احتفال الإكمال.** لمسة واحدة تطلق 4 حركات معاً: `pop-check` + `halo-pulse` للبطاقة + وميض `AmbientPulse` بملء الشاشة + كونفيتي `RewardPop`. مبهج مرة، مرهق سبع مرات يومياً. → احفظ وميض الشاشة الكاملة للإنجازات الكبرى (إكمال اليوم، milestone) فقط.
6. وحّد مقياس الضغط: `.press` يستخدم `scale(0.97)` بينما زر الإكمال `active:scale-90`.

### 2.5 إمكانية الوصول (Accessibility)

1. **لا توجد حالات تركيز للوحة المفاتيح إطلاقاً** — أكبر فجوة. لا `:focus-visible` في globals.css، والأزرار والروابط بلا أي حلقة تركيز. → أضف عالمياً:
   ```css
   :focus-visible { outline: 2px solid var(--color-accent); outline-offset: 2px; }
   ```
2. **فشل تباين `--color-faint`**: `#9b9184` على `#faf6f0` ≈ 2.8:1 (المطلوب 4.5:1) وهو مستخدم لنصوص حقيقية: التاريخ، بذرة اليوم، وسوم الشبكة، تلميحات الخطط. → انقلها لـ`--color-muted`.
3. عنوان حالة `BloomHero` هو `<p>` (سطر 90) وليس heading — أهم حالة في اللوحة خارج مخطط العناوين.
4. الحديقة/الشبكة تنقل المستوى **باللون فقط** — أضف أنماطاً أو نصاً بديلاً.
5. نقاط قوة تُحفظ: `prefers-reduced-motion` مغطى جيداً (globals:347-353)، `aria-label` و`aria-pressed` و`aria-current` منتشرة بوعي.

### 2.6 توحيد الهوية (تغيير جوهري مقترح)

1. **مسار الوردة السباعية معرّف مستقلاً في 4+ ملفات** (`Logo` و`FlowerMark` و`BloomHero` و`HarvestClient` و`GardenGrid`) بإحداثيات مختلفة. → استخرجه لمكوّن SVG أساسي واحد `<Petal />` يضمن أن الزهرة متطابقة في كل مكان.
2. **صراع الهوية: إيموجي ضد أيقونات خطية.** `Icon.tsx` يصف نفسه «بديلٌ فاخر للإيموجي»، لكن أيقونات العادات والأوسمة **إيموجي**. الفاخر والطفولي في نفس الشاشة يلغيان بعضهما. → قرار واحد: إمّا مجموعة أيقونات خطية منسّقة للعادات (الأفخم)، أو احتضان الإيموجي كلياً.
3. على المدى الأبعد: خط عرض (Display) عربي أثقل شخصية يُزاوج مع Almarai لعناوين الهيرو — خط واحد للجسم والعناوين يحرم اللحظات الكبيرة من هيبتها.

## الجزء الثالث: تحسينات البرمجة ⚙️

### 3.1 المصادقة والجلسات
- **الـmiddleware يفحص وجود الكوكي فقط لا صحته** (`src/middleware.ts:6-11`) — إرسال `aoun_session=garbage` يتجاوز التحويل. → استدعِ `jwtVerify` (jose يعمل على الـedge).
- **حارس مصادقة مشترك:** كتلة `getUserId() → 401` منسوخة يدوياً في ~9 مسارات ونُسيت في اثنين (refine وpush DELETE). → غلاف `withAuth(handler)` واحد يمنع هذه الفئة من الأخطاء نهائياً.
- **قناة توقيت لتعداد المستخدمين:** في `login/route.ts:22` لا يُنفَّذ bcrypt عندما لا يوجد البريد، فالرد أسرع بشكل ملحوظ — يكشف وجود الحساب. → قارن دائماً ضد hash وهمي ثابت.
- **JWT لمدة 30 يوماً بلا إبطال**: الخروج يحذف الكوكي فقط والتوكن يبقى صالحاً إن سُرق. → فكّر في jti + قائمة إبطال، أو مدة أقصر مع تجديد.
- سياسة كلمة المرور: 6 أحرف فقط، وbcrypt يقص صامتاً عند 72 بايت. → حد أدنى 8 + حد أقصى 72 برسالة واضحة.
- `/plus` غير محمي وغير موجود في الـmatcher — إن كان تسويقياً فقيّده بوعي، ووسّع الـmatcher ليشمل `/api/*` المحمية.

### 3.2 أخطاء المنطق والاتساق
- **XP غير رتيبة (تتناقص!):** `getDashboard` يمسح 90 يوماً فقط (`HISTORY_DAYS=90`) لحساب `perfectDays` ثم `computeTotalXp(totalCompletions, perfectDays)` حيث الأول **كل التاريخ** والثاني **90 يوماً فقط** — الأيام المثالية القديمة تسقط من الحساب فتنخفض XP مع الوقت. والتحليلات تحسب على 365 يوماً فتظهر أرقام «رحلتك» مختلفة عن اللوحة. → دالة تجميع واحدة مشتركة بنافذة واحدة متسقة.
- **GET يكتب في القاعدة:** `getDashboard` يعمل upsert لـ`UserScore` في كل نداء (`habits.ts:198-204`) والقيمة المخزنة **لا يقرؤها أحد** — عمل ميت وعبء كتابة على كل تحميل صفحة. → احذف الـupsert أو انقله لحدث صريح.
- **التذكيرات لن تعمل أبداً بالإعداد الحالي:** `runReminders` نافذته 5 دقائق (`reminders.ts:60`) بينما cron في `vercel.json` يعمل **مرة يومياً 06:00**. عادة موعدها 17:30 لن تقع في النافذة أبداً. → إمّا cron كل 5 دقائق (`*/5 * * * *`) مع سجل إرسال يمنع التكرار، أو منطق «إرسال دفعة اليوم» صباحاً.
- **سباق حد السبع عادات:** `createHabit` يعمل `count()` ثم `create()` بلا transaction — طلبان متزامنان يتجاوزان الحد. → اجعلها transaction أو قيداً في القاعدة.
- `colorKey @default("mint")` في المخطط (`schema.prisma:58`) **ليست قيمة صالحة** — الصالح: sage/teal/sky/lavender/blush/clay/amber. → صحّح الافتراضي والتعليق.
- `seedOfDay` يستخدم توقيت الخادم المحلي (`messages.ts:109-112`) — تجميلي لكن أصلحه مع 1.3.

### 3.3 طبقة البيانات
- **استعلامات غير محدودة:** `dailyLog.findMany({ where: { userId, completed: true } })` بلا حد تاريخي في `habits.ts:124` و`analytics.ts:57` — يسحب كل التاريخ للذاكرة في كل تحميل ثم يفلتر في JS. → أضف `date: { gte: startKey }` (الفهرس `[userId, date]` موجود أصلاً).
- **N+1 في التذكيرات:** استعلامان لكل مستخدم داخل حلقة، و`sendPushToUser` يستعلم الاشتراكات **لكل عادة مستحقة** (`push.ts:38`) — مستخدم بـ7 عادات = 7 استعلامات اشتراك. → اجلب الاشتراكات مرة لكل مستخدم.
- **JSON كنصوص:** `weekdays` و`microSteps` و`diagnostic` أعمدة `String` تُحلَّل يدوياً بثلاث نسخ من نفس الدالة (`habits.ts:61`، `analytics.ts:32`، `reminders.ts:34`). القاعدة PostgreSQL — → حوّلها لـ`Json` واحذف الـboilerplate.

### 3.4 تصميم الـAPI
- **رموز حالة خاطئة:** «العادة غير موجودة» تُرجع 500 (`habits/[id]/route.ts:19,36`) والصحيح 404؛ وكل أخطاء `createHabit` تُرجع 400 حتى أعطال القاعدة الحقيقية. → خرائط أخطاء دقيقة (404/400/409/500).
- `/api/habits` POST يحلل الجسم **قبل** فحص المصادقة — اعكس الترتيب.
- أضف `code` قابلاً للقراءة آلياً لشكل الخطأ `{ error, code }` بدل مطابقة النصوص العربية في العميل.

### 3.5 المتجر (Zustand) والعمارة
- **كل تعديل يستبدل اللوحة كاملة:** تعديل عنوان عادة واحدة يعيد جلب واستبدال الـ7 كلها، وأي تعديل متفائل على بطاقة أخرى **يُمحى** بالحمولة العائدة — سباق حقيقي عند تعديلين سريعين. → حدّث العادة المعنية فقط من الرد.
- **ازدواج كثيف:** خمس دوال في `store/dashboard.ts` تكرر نفس هيكل «متفائل → fetch → تراجع». → مساعد واحد `mutateHabit()`.
- **حساب streak المتفائل ساذج:** `h.streak + (next ? 1 : -1)` (سطر 98) خاطئ للعادات الأسبوعية، و`xpForCompletion` يُغذّى بـstreak الحساب العام بينما التوثيق يطلب streak العادة (سطر 111) — مفهومان مختلفان يغذيان نفس النافذة الاحتفالية.
- `export { qualifiesForStreak }` من المتجر (سطر 234) — استورده من `@/lib/scoring` مباشرة.
- **كود ميت يُحذف:** `getOrCreateDemoUser` + `DEMO_EMAIL` (`habits.ts:70-106`، لا يُستدعى)، موديل `Reflection` كاملاً (لا قارئ ولا كاتب)، مسار `skip` الميت (يرجع 410 فقط)، ملفات scaffold: `next.svg` و`vercel.svg` و`window.svg` و`file.svg` و`globe.svg`.

### 3.6 الأداء
- كل الصفحات `force-dynamic` بلا أي كاش — تجميعة التحليلات (مسح 365 يوماً) تُعاد في كل زيارة. → `unstable_cache` بمفتاح `userId + dateKey` أو `revalidate` قصيرة.
- `AddHabit` يشحن `HABIT_LIBRARY` كاملة (12 قالباً بخطواتها) في حزمة العميل — مقبول الآن، راقبه مع النمو.
- `WeeklyChart` يحرّك `height` (غير مركّب على GPU) — استخدم `transform: scaleY` مع `transform-origin: bottom`.
- نقاط قوة: جلب اللوحة server-side كـRSC (النمط الصحيح)، `next/font` مع subset عربي وswap.

### 3.7 الـPWA وservice worker
`public/sw.js`:
- **يكيّش الأخطاء:** `c.put(req, copy)` بلا فحص `res.ok` — صفحة 500 قد تُقدَّم من الكاش لاحقاً. → اشترط `res.ok && res.status === 200 && res.type === "basic"`.
- **خصوصية:** HTML المصادَق عليه (`/dashboard` باسم المستخدم وعاداته) يُخزَّن في Cache Storage ويبقى **بعد تسجيل الخروج** — على جهاز مشترك يظهر لشخص آخر. → لا تكيّش المسارات المصادَقة، وامسح الكاش عند الخروج.
- `CACHE = "aoun-v1"` نسخة يدوية لا تُرفع تلقائياً → اربطها بمعرّف البناء.
- الـmanifest: أضف `id` و`screenshots` و`shortcuts`، وطابق `theme_color` مع `layout.tsx` (حالياً `#f7f4ef` ضد `#faf6f0`).

### 3.8 جودة الكود والاختبارات
- `strict: true` جيد لكنه مُبطَل بـ`as` بالضبط حيث تدخل البيانات غير الموثوقة (`(await req.json()) as ...` في كل المسارات) — طبقة zod (بند 1.4) تحل هذا أيضاً.
- **الاختبارات:** ملفان فقط (scoring وrecommend — ممتازان) لكن **صفر اختبارات** لأخطر الوحدات: `date.ts` (موضع خطأ المناطق الزمنية)، حساب نافذة التذكيرات (`inWindow`)، منطق المتجر المتفائل/التراجع، مسارات API. → أضفها بهذا الترتيب.
- `next.config.ts` فارغ: أضف `poweredByHeader: false` وترويسات أمان (CSP، HSTS، X-Frame-Options، Referrer-Policy).
- شغّل `lint` بـ`--max-warnings 0` وأضف `no-floating-promises`.

## ملخص: أهم 10 نقاط لو لم تفعل غيرها

1. سر الجلسات الافتراضي (`auth.ts:10`) — **ثغرة استيلاء كامل**.
2. حارس الـcron fail-closed (`cron/reminders:11`).
3. المنطقة الزمنية الصلبة (`habits.ts:116`, `analytics.ts:49`).
4. طبقة تحقق zod لكل المسارات.
5. مصادقة `refine` وpush DELETE + rate limiting.
6. تأكيد + تراجع للحذف.
7. `:focus-visible` عالمي + تباين `faint`.
8. chevron الرجوع المعكوس وأسهم «←».
9. هدف لمس 44px لزر الإكمال.
10. تدرّج CTA واحد + سلّم خطوط + سلّم زوايا.

**نقاط القوة التي يجب عدم المساس بها:** منطق scoring/recommend النقي المُختبَر، جلب اللوحة كـRSC، transaction الـonboarding، تغطية `prefers-reduced-motion`، طبقة tokens الليلية الكاملة لوضع «الغسق»، نظام الحركة (`--ease-spring`، `BloomHero`، الدخول المتدرج) — هذا هو ما يميز المنتج.

الفجوة بين الوضع الحالي ومنتج «فاخر» حقاً هي **الانضباط النظامي**: تدرّج واحد، سلّم واحد، نظام تغذية راجعة واحد — مع سد ثغرات الأمان والوصول.
